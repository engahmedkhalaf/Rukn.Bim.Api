How It Works

5D BOQ Workflow

From model export to BOQ code generation in five simple steps.

1

Export Model Elements

Select your scope (Current Selection, Active View, or Full Model), apply optional category/family/type filters, and export all elements to an Excel workbook with Revit Elements and BOQ Mapping sheets.

2

Map BOQ Codes in Excel

Fill in the mapping columns: PACKAGE_NO, BILL_NO, SYSTEM_CODE, PAGE_NO, ITEM_NO, plus optional fields like DISTRICT_CODE, ASSET_GROUP, and more.

3

Import Mapped Excel

Load the completed Excel file back into the add-in. The smart parser auto-detects column headers with flexible naming support (e.g. "Package No" or "PACKAGE_NO").

4

Validate Mapping

Run a dry validation to preview matched/unmatched elements before committing any changes. Review warnings for incomplete BOQ records.

5

Generate 5D BOQ Codes

One click creates all 14 shared parameters, matches elements, and writes values including the composite QIC_5D_BOQ CODE (e.g. PKG01-B02-MEP-P05-I003). Export an audit report for QA.
