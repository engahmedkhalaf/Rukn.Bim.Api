📊 Excel Format Reference

Column specifications and mapping template structure

Worksheet Detection

The parser looks for worksheets in this order:

  1. Revit Elements — primary sheet name
  2. BOQ Mapping — alternative sheet name
  3. First worksheet — fallback if neither name is found

Required Columns

At minimum, the Excel file must contain one identifier column plus the 5 BOQ code fields:

Column HeaderAlternate NamesDescription
Type NameTYPE NAME, TYPEElement type name for matching
Element IDELEMENT ID, ELEMENTIDRevit Element ID for exact matching
Package NoPACKAGE NO, PACKAGE_NOPackage number
Bill NoBILL NO, BILL_NOBill number
System CodeSYSTEM CODE, SYSTEM_CODESystem classification code
Page NoPAGE NO, PAGE_NOPage reference number
Item NoITEM NO, ITEM_NOItem reference number
ℹ️ Identifier Requirement

At least one of Type Name or Element ID must be present. Both can coexist — Element ID takes priority during matching.

Header Detection

  • Headers are read from Row 1 only
  • Matching is case-insensitive (e.g. "package no" = "PACKAGE NO")
  • The parser scans up to 200 columns
  • Scanning stops after 10 consecutive empty header cells

Data Rows

  • Data starts from Row 2
  • Supports up to 100,000 rows
  • Parsing stops after 20 consecutive empty rows
  • A row must have either Type Name or Element ID to be imported

Exported File Structure

When you click Export Elements, the output Excel contains:

Sheet 1: Revit Elements

Element ID | Unique ID | Category | Family Name | Type Name | Level | Workset | Mark | Package No | Bill No | System Code | Page No | Item No | QIC_5D_BOQ CODE

Sheet 2: BOQ Mapping (Empty Template)

Category | Family Name | Type Name | Package No | Bill No | System Code | Page No | Item No | District Code | Asset Group | Asset Type | Location Code | Description | ABS L1 | ABS L2 | ABS L3
💡 Workflow Tip

Export first, then copy unique Category/Family/Type combinations to the BOQ Mapping sheet and fill in your codes there.
